# `/idea`

Use this command when a feature request is still exploratory and belongs in [`specs/ideas`](../../specs/ideas/README.md) rather than a numbered promoted spec.

## Goals

- capture the user value and rough workflow
- record open questions and unknowns
- define what would make the idea ready for promotion

## Command Checklist

1. Review [`.specify/memory/constitution.md`](../memory/constitution.md) for repository planning constraints.
2. Review [`specs/ideas/README.md`](../../specs/ideas/README.md) for incubator rules.
3. Start from [`.specify/templates/idea-template.md`](../templates/idea-template.md).
4. Create or update one Markdown file in `specs/ideas/` for the concept.
5. Add or update breadcrumbs and linked navigation where practical.
6. Keep the content exploratory; do not force implementation decisions that are not yet settled.

## Expected Output

A Markdown idea note that explains the value, rough user journey, open questions, and promotion criteria for the feature.
