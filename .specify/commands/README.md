# Spec Commands

This directory stores optional repo-local planning command prompts for tools that support command-style workflows.

These files are documentation-first helpers layered on top of the templates in [`.specify/templates`](../templates/README.md). If your tooling does not load repo-local commands automatically, use these files as reusable prompt/checklist references.

## Files in this directory

- [`idea.md`](../../.specify/commands/idea.md) — incubate or refine a rough feature idea in [`specs/ideas`](../../specs/ideas/README.md).
- [`spec.md`](../../.specify/commands/spec.md) — promote a clear idea or feature request into a numbered spec under [`specs`](../../specs/README.md).
- [`plan.md`](../../.specify/commands/plan.md) — break a promoted spec into implementation sequencing and delivery notes.
