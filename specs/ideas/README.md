# Ideas

This directory is the incubator for new features before they become promoted specs.

## Files in this directory

Keep ideas in Markdown only. Prefer one file per concept and use [`.specify/templates/idea-template.md`](../../.specify/templates/idea-template.md) as the starting point.

## Rules

- Treat idea files as exploratory notes, not committed implementation scope.
- Capture value, rough user flow, open questions, and promotion criteria.
- Promote an idea into `specs/###-feature-name/` only when scope and acceptance are clear enough to plan.
- Link deferred follow-up work back here when a promoted spec intentionally leaves ideas for later.
